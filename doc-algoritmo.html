Sistema de Asignación Óptima de Tareas

Explicación técnica, arquitectura y funcionamiento del sistema de optimización basado en ramificación y poda.

arrow_back Volver code Code

Introducción y Propósito

El Sistema de Asignación Óptima de Tareas es una aplicación Java que resuelve el problema clásico de asignación de tareas a agentes, minimizando el costo total mediante un algoritmo de ramificación y poda (branch-and-bound).

  • • Optimización: Encontrar la asignación de menor costo posible
  • • Eficiencia: Resolver problemas de tamaño medio en tiempos razonables
  • • Robustez: Manejo profesional de errores y validaciones
  • • Trazabilidad: Logging detallado y estadísticas de rendimiento
  • • Usabilidad: Interfaz simple tanto por archivo como interactiva

Descripción del Problema

Dado n agentes y n tareas con una matriz de costos C[i][j], el objetivo es encontrar una asignación uno-a-uno que minimice el costo total.

Agentes: [Trabajador1, Trabajador2, Trabajador3]
Tareas:  [TareaA, TareaB, TareaC]

Matriz de Costos:
      TareaA TareaB TareaC
Trab1   10     15     20
Trab2   12     10     18
Trab3   15     18     12

Solución Óptima:
- Trabajador1 → TareaA (costo: 10)
- Trabajador2 → TareaB (costo: 10)
- Trabajador3 → TareaC (costo: 12)
COSTO TOTAL: 32

Algoritmo Implementado

Ramificación y Poda (Branch and Bound)

  • • Estructura del Árbol de Decisión: Se exploran todas las asignaciones posibles, podando ramas no óptimas.
  • • Función de Cota: Calcula el coste mínimo posible para cada nodo parcial.
  • • Estrategia de Poda: Si la cota de un nodo es mayor o igual al mejor coste conocido, se descarta esa rama.
  • • Optimizaciones: Heap binario, precálculo de cotas, validaciones y poda anticipada.

Arquitectura y Componentes

El sistema sigue un patrón Facade + Command + Observer con separación clara de responsabilidades.

  • • TaskAssignmentSystem: Orquestador principal
  • • Game: Motor del algoritmo
  • • UserInterface: Interfaz de usuario
  • • Heap, Node, GameData, Matrix, Vector, VectorB: Estructuras de datos especializadas

Ejemplo de Uso

Entrada:
3 3
10 15 20
12 10 18
15 18 12

Salida:
Solución Óptima:
1 → 1 (costo: 10)
2 → 2 (costo: 10)
3 → 3 (costo: 12)
Costo Total: 32
